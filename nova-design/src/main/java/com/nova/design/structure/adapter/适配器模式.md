# 适配器模式

将一个类的接口转换成客户希望的另外一个接口，使得原本由于接口不兼容而不能一起工作的那些类能一起工作。

角色：

1. 目标接口（Target）：客户所期待的接口。可以是具体类 或 抽象的类，也可以是接口
2. 需要适配的类（Adapter）
3. 适配器（Adapter）：通过包装一个需要适配的对象，把原接口转换成目标接口

区别：

- 类适配器：单继承，一次最多只能适配一个适配者类
- 对象适配器：可以把多个不同的适配者适配到同一个目标

> tips: 推荐使用对象适配器
